## foreignKey 配置

### order 表 schema

```javascript
// database/order.schema.json
{
  "bsonType": "object",
  "properties": {
    "book_id": {
      "bsonType": "string",
      "foreignKey": "book._id"  // 关联到 book 表的 _id
    },
    "quantity": {
      "bsonType": "int"
    }
  }
}
```

### book 表 schema

```javascript
// database/book.schema.json
{
  "bsonType": "object",
  "properties": {
    "title": {
      "bsonType": "string"
    },
    "author": {
      "bsonType": "string"
    },
    "price": {
      "bsonType": "int"
    }
  }
}
```

## 虚拟联表查询

### 级别1: 直接联表

```javascript
const db = uniCloud.databaseForJQL()
const res = await db.collection('order,book').get()

// 返回结果: order 表的数据,book_id 字段会被替换成 book 表的记录
{
  "data": [{
    "_id": "order-id",
    "book_id": [{
      "_id": "book-id",
      "title": "三国演义",
      "author": "罗贯中"
    }],
    "quantity": 100
  }]
}
```

### 级别2: 带条件联表

```javascript
// 查询书名为"三国演义"的订单
const res = await db.collection('order,book')
  .where('book_id.title == "三国演义"')
  .get()
```

### 级别3: 带字段过滤

**写法1: 使用 {} (推荐)**
```javascript
const res = await db.collection('order,book')
  .field('book_id{title,author},quantity')
  .get()
```

**写法2: 使用点号 (HBuilderX 3.3.7+)**
```javascript
const res = await db.collection('order,book')
  .field('book_id.title,book_id.author,quantity')
  .get()
```

### 级别4: 字段别名

```javascript
const res = await db.collection('order,book')
  .field('book_id{title as book_title,author as book_author},quantity as order_quantity')
  .orderBy('order_quantity desc')
  .get()

// 返回结果
{
  "data": [{
    "book_id": [{
      "book_title": "三国演义",
      "book_author": "罗贯中"
    }],
    "order_quantity": 555
  }]
}
```

## 临时表联表查询

### 性能优化对比

**❌ 不推荐: 先联表再过滤**
```javascript
// 会先将所有数据进行关联,性能差
const res = await db.collection('order,book')
  .where('book_id.title == "三国演义" && quantity > 100')
  .get()
```

**✅ 推荐: 先过滤再联表**
```javascript
// 先对主表和副表进行过滤,然后再联表,性能好
const order = db.collection('order')
  .where('quantity > 100')
  .field('book_id,quantity')
  .getTemp();  // 注意是 getTemp 不是 get

const book = db.collection('book')
  .where('title == "三国演义"')
  .field('_id,title,author')
  .getTemp();

const res = await db.collection(order, book).get();
```

### 临时表方法限制

临时表内可以使用的方法(必须按顺序调用):
```javascript
where
field
orderBy
skip
limit
getTemp
```

### 组合虚拟联表方法限制

组合虚拟联表可以使用的方法(必须按顺序调用):
```javascript
foreignKey  // HBuilderX 3.3.7+
where
field
groupBy    // HBuilderX 3.4.0+
groupField // HBuilderX 3.4.0+
distinct   // HBuilderX 3.4.0+
orderBy
skip
limit
get
```

## 副表 foreignKey 联查

### 双向关联示例

**comment 表 schema:**
```javascript
{
  "properties": {
    "article": {
      "bsonType": "string",
      "foreignKey": "article.article_id"
    },
    "sender": {
      "bsonType": "string",
      "foreignKey": "user.uid"
    },
    "receiver": {
      "bsonType": "string",
      "foreignKey": "user.uid"
    }
  }
}
```

### 使用 foreignKey 方法指定关联

```javascript
const comment = db.collection('comment')
  .where('comment_id == "1-1"')
  .getTemp();

const user = db.collection('user')
  .field('uid,name')
  .getTemp();

// 只使用 comment 表 receiver 字段的 foreignKey
db.collection(comment, user)
  .foreignKey('comment.receiver')
  .get()
```

### 副表 foreignKey 返回结构

**主表某字段 foreignKey 指向副表时:**
```javascript
{
  "主表字段名1": "xxx",
  "主表内foreignKey指向副表的字段名": [{
    "副表字段名1": "xxx",
    "副表字段名2": "xxx"
  }]
}
```

**副表某字段 foreignKey 指向主表时:**
```javascript
{
  "主表字段名1": "xxx",
  "主表内被副表foreignKey指向的字段名": {
    "副表表名": [{
      "副表字段名1": "xxx",
      "副表字段名2": "xxx"
    }],
    "_value": "主表字段原始值"
  }
}
```

## 联表查询注意事项

### 1. field 必须包含关联字段

```javascript
// ❌ 错误: field 中不包含关联字段,无法建立关联
const order = db.collection('order')
  .field('quantity')  // 缺少 book_id
  .getTemp();

// ✅ 正确: field 中包含关联字段
const order = db.collection('order')
  .field('book_id,quantity')  // 包含 book_id
  .getTemp();
```

### 2. 权限校验

```javascript
// ❌ 错误: 对虚拟联表过滤,无法通过权限校验
const order = db.collection('order').getTemp();
const res = await db.collection(order, 'book')
  .where('uid == $cloudEnv_uid')  // 无法通过 order 表的权限校验
  .get();

// ✅ 正确: 先对主表过滤,再联表
const order = db.collection('order')
  .where('uid == $cloudEnv_uid')  // 先过滤,满足权限
  .getTemp();
const res = await db.collection(order, 'book').get();
```

### 3. 多表联表

```javascript
// 支持3个、4个表联表查询
const table1 = db.collection('table1').getTemp();
const table2 = db.collection('table2').getTemp();
const table3 = db.collection('table3').getTemp();

db.collection(table1, table2, table3).get();
```

### 4. 不支持副表与副表之间的关联

```javascript
// 只能使用副表与主表之间的关联关系
// 不可使用副表与副表之间的关联关系
```

## 实战案例

### 案例1: 订单列表页面

```javascript
// 需求: 显示订单列表,包含商品名称和作者
// 实现: 使用 JQL 联表查询

const db = uniCloud.databaseForJQL()
const order = db.collection('order')
  .field('book_id,quantity,create_time')
  .orderBy('create_time desc')
  .getTemp()
const book = db.collection('book')
  .field('_id,title,author,price')
  .getTemp()

const res = await db.collection(order, book)
  .skip((pageIndex - 1) * pageSize)
  .limit(pageSize)
  .get({ getCount: true })

return {
  list: res.data,
  total: res.count
}
```

### 案例2: 用户订单查询(子查询)

```javascript
// 需求: 查询指定公司的员工订单
// 实现: 使用 JQL 子查询

const db = uniCloud.databaseForJQL()

// 先查询符合条件的公司
const companyFilter = db.collection('company')
  .where('status == 0')
  .field('_id')
  .getTemp()

// 再查询员工订单
const res = await db.collection('employee')
  .where(`register_date > 1670000000000 && company_id in ${companyFilter}`)
  .get()

return res.data
```
