## 性能优化建议

### 1. 联表查询优化

```javascript
// ❌ 不推荐: 先联表再过滤
db.collection('order,book')
  .where('book_id.title == "三国演义" && quantity > 100')
  .get()

// ✅ 推荐: 先过滤再联表
const order = db.collection('order')
  .where('quantity > 100')
  .getTemp()
const book = db.collection('book')
  .where('title == "三国演义"')
  .getTemp()
db.collection(order, book).get()
```

### 2. 字段过滤优化

```javascript
// ❌ 不推荐: 查询所有字段
db.collection('user').get()

// ✅ 推荐: 只查询需要的字段
db.collection('user')
  .field('name,avatar,email')
  .get()
```

### 3. 分页查询优化

```javascript
// ❌ 不推荐: 一次性加载所有数据
const allData = await db.collection('user').get()

// ✅ 推荐: 使用分页加载
const pageSize = 20
const page1 = await db.collection('user')
  .skip(0)
  .limit(pageSize)
  .get()
```

### 4. 索引优化

```javascript
// 在 web 控制台为常用查询字段创建索引
// 1. where 条件字段
// 2. orderBy 排序字段
// 3. foreignKey 关联字段

// 示例: 为 user 表的 mobile 字段创建索引
// web控制台: 数据库 > user表 > 索引管理 > 添加索引
// 字段: mobile, 类型: 升序/降序
```

### 5. 使用 multiSend 合并请求

```javascript
// ❌ 不推荐: 分开请求
const banner = await db.collection('banner').get()
const notice = await db.collection('notice').get()

// ✅ 推荐: 合并请求
const bannerQuery = db.collection('banner').getTemp()
const noticeQuery = db.collection('notice').getTemp()
const res = await db.multiSend(bannerQuery, noticeQuery)
```

### 6. 避免使用正则查询的前缀通配符

```javascript
// ❌ 不推荐: 正则以通配符开头,无法使用索引
db.collection('user').where('/^test/.test(name)').get()

// ✅ 推荐: 正则不以通配符开头,可以使用索引
db.collection('user').where('/^test/.test(name)').get()
```

## 常见错误处理

### 1. 权限错误(PERMISSION_ERROR)

```javascript
// 错误: 权限校验未通过
// 解决: 检查 schema 的 permission 配置

// schema 配置
{
  "permission": {
    "read": "doc.uid == auth.uid",
    "update": "doc.uid == auth.uid"
  }
}

// 客户端调用
const db = uniCloud.databaseForJQL()
db.collection('user')
  .where('uid == $cloudEnv_uid')  // 必须包含此条件
  .get()
```

### 2. 语法错误(SYNTAX_ERROR)

```javascript
// ❌ 错误: 运算符两侧不能都是数据库字段(简单查询)
db.collection('user').where('field1 == field2').get()

// ✅ 正确: 使用复杂查询条件
db.collection('user').where('field1 > field2').get()
```

### 3. 数据格式错误(VALIDATION_ERROR)

```javascript
// 错误: 数据格式不符合 schema 规则
// 解决: 检查 schema 的 validator 配置

// schema 配置
{
  "properties": {
    "email": {
      "bsonType": "string",
      "pattern": "^[\\w-]+(\\.[\\w-]+)*@[\\w-]+(\\.[\\w-]+)+$"
    }
  }
}

// 客户端调用
try {
  await db.collection('user')
    .add({
      email: 'invalid-email'  // 会报错
    })
} catch (err) {
  console.error(err.errMsg)  // 数据格式错误
}
```

### 4. 索引冲突(DUPLICATE_KEY)

```javascript
// 错误: 唯一索引字段重复
// 解决: 检查是否已存在相同数据

// 先查询是否存在
const exist = await db.collection('user')
  .where('mobile == "13800138000"')
  .get()

if (exist.data.length > 0) {
  console.log('手机号已存在')
} else {
  // 再新增
  await db.collection('user')
    .add({
      mobile: '13800138000'
    })
}
```

### 5. Token 相关错误

```javascript
// TOKEN_INVALID_TOKEN_EXPIRED: token已过期
// 解决: 重新登录获取新token

// TOKEN_INVALID_WRONG_TOKEN: token校验未通过
// 解决: 检查token是否正确

// TOKEN_INVALID_ANONYMOUS_USER: 当前用户为匿名用户
// 解决: 需要登录后才能操作
```

## JQL vs vk.baseDao 选择指南

### 使用 JQL 的场景

- ✅ 客户端直接查询数据库(clientDB)
- ✅ 需要简单的类 SQL 语法
- ✅ 使用 foreignKey 进行联表查询
- ✅ 需要树形查询、分组统计等高级功能
- ✅ 利用 DB Schema 的权限校验
- ✅ 使用 `<unicloud-db>` 组件

### 使用 vk.baseDao 的场景

- ✅ 云函数中进行复杂的数据处理
- ✅ 需要使用 MongoDB 的聚合管道
- ✅ 需要使用数据库事务
- ✅ 需要更细粒度的性能控制
- ✅ 已有 vk 框架的项目
- ✅ 需要使用 vk 的内置功能(如自动生成 _add_time)

### 对比表格

| 特性 | JQL | vk.baseDao |
|------|-----|------------|
| 使用场景 | 客户端+云函数 | 仅云函数 |
| 语法 | 类 SQL 字符串 | JSON 对象 |
| 联表查询 | 通过 foreignKey 自动关联 | 需要手动编写 lookup |
| 权限校验 | 自动进行 schema 权限校验 | 需要手动编写权限逻辑 |
| 学习成本 | 低,类似 SQL | 中,需要了解 MongoDB |
| 性能 | 适合常规查询 | 适合复杂查询 |
| 事务支持 | 支持(HBuilderX 4.81+) | 支持 |

## 实战案例

### 案例1: 订单列表页面(联表查询)

```javascript
// 需求: 显示订单列表,包含商品名称和作者
// 实现: 使用 JQL 联表查询

const db = uniCloud.databaseForJQL()
const order = db.collection('order')
  .field('book_id,quantity,create_time')
  .orderBy('create_time desc')
  .getTemp()
const book = db.collection('book')
  .field('_id,title,author,price')
  .getTemp()

const res = await db.collection(order, book)
  .skip((pageIndex - 1) * pageSize)
  .limit(pageSize)
  .get({ getCount: true })

return {
  list: res.data,
  total: res.count
}
```

### 案例2: 数据统计报表(分组统计)

```javascript
// 需求: 统计每日订单数量和金额
// 实现: 使用 JQL 分组统计

const db = uniCloud.databaseForJQL()
const res = await db.collection('order')
  .groupBy('dateToString(add(new Date(0),create_time),"%Y-%m-%d","+0800") as date')
  .groupField('count(*) as order_count,sum(total_amount) as total_amount')
  .orderBy('date asc')
  .get()

return res.data
```

### 案例3: 部门树形结构(树形查询)

```javascript
// 需求: 显示部门树形结构
// 实现: 使用 JQL 树形查询

const db = uniCloud.databaseForJQL()
const res = await db.collection('department')
  .where('status == 0')
  .get({
    getTree: {
      limitLevel: 10
    }
  })

return res.data
```

### 案例4: 商品搜索(正则查询)

```javascript
// 需求: 搜索商品名称
// 实现: 使用 JQL 正则查询

const db = uniCloud.databaseForJQL()
const searchVal = '手机'

const res = await db.collection('goods')
  .where(`${new RegExp(searchVal, 'i')}.test(name)`)
  .orderBy('sales desc')
  .skip(0)
  .limit(20)
  .get({ getCount: true })

return {
  list: res.data,
  total: res.count
}
```

### 案例5: 用户订单查询(子查询)

```javascript
// 需求: 查询指定公司的员工订单
// 实现: 使用 JQL 子查询

const db = uniCloud.databaseForJQL()

// 先查询符合条件的公司
const companyFilter = db.collection('company')
  .where('status == 0')
  .field('_id')
  .getTemp()

// 再查询员工订单
const res = await db.collection('employee')
  .where(`register_date > 1670000000000 && company_id in ${companyFilter}`)
  .get()

return res.data
```

## 安全建议

### 1. 权限最小化原则

```javascript
// 只给用户必要的权限
{
  "permission": {
    "read": "doc.uid == auth.uid",  // 只能读自己的数据
    "create": false,                 // 禁止新增
    "update": false,                 // 禁止修改
    "delete": false                  // 禁止删除
  }
}
```

### 2. 敏感字段保护

```javascript
// 敏感字段禁止读取和写入
{
  "properties": {
    "password": {
      "bsonType": "string",
      "permission": {
        "read": false,  // 禁止读取
        "write": false  // 禁止写入
      }
    }
  }
}
```

### 3. 输入数据校验

```javascript
// 在 schema 中配置数据校验规则
{
  "properties": {
    "mobile": {
      "bsonType": "string",
      "pattern": "^1[3-9]\\d{9}$",  // 手机号格式校验
      "maxLength": 11
    }
  }
}
```

### 4. 防止SQL注入

```javascript
// JQL 会自动处理参数化查询,防止SQL注入
// 不要使用字符串拼接的方式构造查询条件

// ❌ 不推荐: 字符串拼接
const name = getUserInput()
db.collection('user').where(`name == "${name}"`).get()

// ✅ 推荐: 使用参数化查询
const name = getUserInput()
db.collection('user').where({
  name: name
}).get()
```
