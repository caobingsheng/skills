## 树形查询 getTree

### Schema 配置

```javascript
// database/department.schema.json
{
  "bsonType": "object",
  "properties": {
    "name": {
      "bsonType": "string"
    },
    "parent_id": {
      "bsonType": "string",
      "parentKey": "_id"  // 指定父子关系
    },
    "status": {
      "bsonType": "int"
    }
  }
}
```

### 级别1: 查询整棵树

```javascript
const db = uniCloud.databaseForJQL()
const res = await db.collection('department')
  .get({
    getTree: true
  })

// 返回结果
{
  "data": [{
    "_id": "1",
    "name": "总部",
    "parent_id": "",
    "children": [{
      "_id": "11",
      "name": "一级部门A",
      "parent_id": "1",
      "children": []
    }]
  }]
}
```

### 级别2: 限制层级

```javascript
const res = await db.collection('department')
  .get({
    getTree: {
      limitLevel: 3  // 最多查询3级(不包含当前层级)
    }
  })
```

### 级别3: 指定起始节点

```javascript
const res = await db.collection('department')
  .where('status == 0')
  .get({
    getTree: {
      startWith: '_id=="1"'  // 从_id为1的节点开始查询
    }
  })
```

### getTree 参数说明

- `limitLevel`: 最大查询层级(不包含当前层级),默认10,最大15,最小1
- `startWith`: 第一层级条件,默认 `'parent_id==null||parent_id==""'`

### 注意事项

- 暂不支持使用 getTree 的同时使用联表查询
- 如果使用了 where 条件会对所有查询的节点生效
- 如果使用了 limit 设置最大返回数量仅对根节点生效

## 树形路径查询 getTreePath

### 查询父节点路径(面包屑场景)

```javascript
const db = uniCloud.databaseForJQL()
const res = await db.collection('department')
  .get({
    getTreePath: {
      startWith: '_id=="5fe77232974b6900018c6cb1"'
    }
  })

// 返回从根节点到目标节点的完整路径
{
  "data": [{
    "_id": "1",
    "name": "总部",
    "parent_id": "",
    "children": [{
      "_id": "11",
      "name": "一级部门A",
      "parent_id": "1"
    }]
  }]
}
```

### getTreePath 参数说明

- `limitLevel`: 最大查询层级(不包含当前层级),默认10,最大15,最小1
- `startWith`: 末级节点的条件,此参数不可以省略

### 注意事项

- 暂不支持使用 getTreePath 的同时使用其他联表查询语法
- 如果使用了 where 条件会对所有查询的节点生效

## 分组统计 groupBy

### 级别1: 求和统计

```javascript
// 场景: 统计每个班级的总分
const res = await db.collection('score')
  .groupBy('grade,class')
  .groupField('sum(score) as totalScore')
  .get()

// 返回结果
{
  "data": [{
    "grade": "1",
    "class": "A",
    "totalScore": 20
  }, {
    "grade": "1",
    "class": "B",
    "totalScore": 40
  }]
}
```

### 级别2: 求均值统计

```javascript
// 场景: 统计每个班级的平均分
const res = await db.collection('score')
  .groupBy('grade,class')
  .groupField('avg(score) as avgScore')
  .get()
```

### 级别3: 计数统计

```javascript
// 场景: 统计每个班级的人数
const res = await db.collection('score')
  .groupBy('grade,class')
  .groupField('count(*) as totalStudents')
  .get()
```

### 级别4: 按日分组统计

```javascript
// 场景: 统计每日新增用户数量
const res = await db.collection('uni-id-users')
  .groupBy('dateToString(add(new Date(0),register_date),"%Y-%m-%d","+0800") as date')
  .groupField('count(*) as newusercount')
  .get()

// 返回结果
{
  "data": [{
    "date": "2021-01-23",
    "newusercount": 3
  }, {
    "date": "2021-01-22",
    "newusercount": 2
  }]
}
```

### 使用 field 预处理

```javascript
// 场景: score 是数组,先求和再分组
const res = await db.collection('score')
  .field('grade,class,sum(score) as userTotalScore')
  .groupBy('grade,class')
  .groupField('avg(userTotalScore) as avgScore')
  .get()
```

### 注意事项

- groupField 内只能使用分组运算方法(count, sum, avg 等)
- 使用 groupField 时返回结果不会默认包含 `_id` 字段
- 不应该在 groupBy 和 groupField 里使用 `_id` 字段

## 数据去重 distinct

### 级别1: 单字段去重

```javascript
const res = await db.collection('score')
  .field('grade')
  .distinct()
  .get()

// 返回所有不重复的 grade
{
  "data": [{
    "grade": "1"
  }, {
    "grade": "2"
  }]
}
```

### 级别2: 多字段去重

```javascript
// 获取所有参赛班级(grade 和 class 组合去重)
const res = await db.collection('score')
  .field('grade,class')
  .distinct()
  .get()

// 返回结果
{
  "data": [{
    "grade": "1",
    "class": "A"
  }, {
    "grade": "1",
    "class": "B"
  }, {
    "grade": "2",
    "class": "A"
  }]
}
```

### 级别3: 带条件去重

```javascript
const res = await db.collection('score')
  .field('grade,class')
  .where('score > 10')
  .distinct()
  .get()
```

### 注意事项

- distinct 指对返回结果中完全相同的记录进行去重
- 因为 `_id` 字段是必然不同的,所以使用 distinct 时必须同时指定 field
- field 中不可存在 `_id` 字段

## 地理位置查询 geoNear

### LBS 场景: 查询附近500米内的POI

```javascript
const db = uniCloud.databaseForJQL()
const res = await db.collection('poi')
  .aggregate()
  .geoNear({
    distanceField: 'distance',        // 存放距离的字段名
    spherical: true,                   // 必填,值为 true
    near: new db.Geo.Point(116.397689, 39.904626),  // 查询中心点
    maxDistance: 500,                  // 最大距离(米)
    query: 'status == 1'               // 额外的查询条件
  })
  .end()
```

### geoNear 参数说明

- `near`: GeoPoint,用于判断距离的点(必填)
- `spherical`: 必须为 true(必填)
- `maxDistance`: 距离最大值(可选)
- `minDistance`: 距离最小值(可选)
- `query`: 要求记录必须同时满足的条件(可选)
- `distanceMultiplier`: 返回时在距离上乘以该数字(可选)
- `distanceField`: 存放距离的输出字段名(必填)
- `includeLocs`: 列出要用于距离计算的字段(可选)
- `key`: 选择要用的地理位置索引(可选)

### 注意事项

- 存在 geoNear 时其 query 参数将取代 where/doc 作为权限校验依据
- 新增于 HBuilderX 3.6.10

## 实战案例

### 案例1: 部门树形结构

```javascript
// 需求: 显示部门树形结构
// 实现: 使用 JQL 树形查询

const db = uniCloud.databaseForJQL()
const res = await db.collection('department')
  .where('status == 0')
  .get({
    getTree: {
      limitLevel: 10
    }
  })

return res.data
```

### 案例2: 数据统计报表

```javascript
// 需求: 统计每日订单数量和金额
// 实现: 使用 JQL 分组统计

const db = uniCloud.databaseForJQL()
const res = await db.collection('order')
  .groupBy('dateToString(add(new Date(0),create_time),"%Y-%m-%d","+0800") as date')
  .groupField('count(*) as order_count,sum(total_amount) as total_amount')
  .orderBy('date asc')
  .get()

return res.data
```

### 案例3: 商品搜索(正则查询)

```javascript
// 需求: 搜索商品名称
// 实现: 使用 JQL 正则查询

const db = uniCloud.databaseForJQL()
const searchVal = '手机'

const res = await db.collection('goods')
  .where(`${new RegExp(searchVal, 'i')}.test(name)`)
  .orderBy('sales desc')
  .skip(0)
  .limit(20)
  .get({ getCount: true })

return {
  list: res.data,
  total: res.count
}
```
