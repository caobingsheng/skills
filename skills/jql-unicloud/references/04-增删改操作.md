## 新增数据 add

### 级别1: 单条新增

```javascript
const db = uniCloud.databaseForJQL()
const res = await db.collection('user')
  .add({
    name: '张三',
    age: 18,
    email: 'test@example.com'
  })

// 返回结果
{
  "id": "new-record-id",  // 新增记录的_id
  "errCode": 0
}
```

### 级别2: 带错误处理

```javascript
try {
  const res = await db.collection('user')
    .add({
      name: '张三',
      age: 18
    })

  uni.showToast({
    title: '新增成功',
    icon: 'success'
  })
} catch (err) {
  uni.showModal({
    content: err.message || '新增失败',
    showCancel: false
  })
}
```

### 级别3: 批量新增

```javascript
const res = await db.collection('user')
  .add([
    { name: '张三', age: 18 },
    { name: '李四', age: 19 },
    { name: '王五', age: 20 }
  ])

// 返回结果
{
  "ids": ["id1", "id2", "id3"],  // 批量新增所有记录的_id
  "inserted": 3,                 // 新增成功的条数
  "errCode": 0
}
```

### 注意事项

- data 中不需要包括 `_id` 字段,数据库会自动维护
- 如果是非 admin 账户新增数据,需要在 DB Schema 中配置 permission 的 create 权限
- 阿里云: 若集合表不存在,调用 add 方法会自动创建集合表

## 更新数据 update

### doc 方式更新(精确更新)

```javascript
const db = uniCloud.databaseForJQL()
const res = await db.collection('user')
  .doc('doc-id')
  .update({
    name: '新名字',
    age: 20
  })

// 返回结果
{
  "updated": 1,  // 更新成功的条数
  "errCode": 0
}
```

### where 方式更新(批量更新)

```javascript
// 场景: 批量修改状态
const res = await db.collection('user')
  .where('age < 18')
  .update({
    status: 'underage'
  })

// 返回结果
{
  "updated": 10,  // 更新成功的条数
  "errCode": 0
}
```

### 更新嵌套字段

```javascript
// 场景1: 更新嵌套对象
await db.collection('user')
  .doc('doc-id')
  .update({
    'count.fav': 1  // 更新嵌套对象
  })

// 更新前
{
  "count": {
    "fav": 0,
    "follow": 0
  }
}

// 更新后
{
  "count": {
    "fav": 1,
    "follow": 0
  }
}
```

### 更新数组元素

```javascript
// 场景2: 更新数组指定下标的元素
await db.collection('user')
  .doc('doc-id')
  .update({
    'arr.1': 'newValue'  // 更新数组下标为1的元素
  })

// 更新前
{
  "arr": ["hello", "world"]
}

// 更新后
{
  "arr": ["hello", "newValue"]
}
```

### 注意事项

- 如果是非 admin 账户修改数据,需要在 DB Schema 中配置 permission 的 update 权限
- 更新数据时键值不可使用 `{'a.b.c': 1}` 的形式,需要写成 `{a:{b:{c:1}}}` 形式
- 禁止使用更新操作符 `db.command.inc` 等

## 删除数据 remove

### doc 方式删除(精确删除)

```javascript
const db = uniCloud.databaseForJQL()
await db.collection('user')
  .doc('doc-id')
  .remove()

// 返回结果
{
  "deleted": 1,  // 删除的条数
  "errCode": 0
}
```

### where 方式删除(批量删除)

```javascript
// 场景: 删除所有测试数据
await db.collection('user')
  .where('isTest == true')
  .remove()

// 返回结果
{
  "deleted": 5,  // 删除的条数
  "errCode": 0
}
```

### 删除所有数据(危险操作)

```javascript
// ⚠️ 警告: 会删除表中所有数据
const dbCmd = db.command
await db.collection('user')
  .where({ _id: dbCmd.neq(null) })
  .remove()
```

### 注意事项

- 如果是非 admin 账户删除数据,需要在 DB Schema 中配置 permission 的 delete 权限
- 数据量很多的情况下删除会超时,但是数据仍会全部删除掉

## 事务操作

### 基础事务(数据一致性场景)

```javascript
const dbJQL = uniCloud.databaseForJQL()
const transaction = await dbJQL.startTransaction()

try {
  // 操作1: 扣减库存
  await transaction.collection('goods')
    .doc('goods-id')
    .update({ stock: -1 })

  // 操作2: 创建订单
  await transaction.collection('order')
    .add({
      goods_id: 'goods-id',
      quantity: 1
    })

  await transaction.commit()  // 提交事务
  console.log('事务执行成功')
} catch (e) {
  await transaction.rollback()  // 回滚事务
  console.error('事务执行失败', e)
}
```

### 事务使用场景

- 银行转账: 一个账户扣款,另一个账户收款
- 下单流程: 扣减库存 + 创建订单
- 数据一致性: 多个操作要么全部成功,要么全部失败

### 注意事项

- 新增于 HBuilderX 4.81
- JQL 事务不支持 `doc.set`、`where.updateAndReturn` 方法
- JQL 事务中 `update` 不支持使用更新操作符
- JQL 触发器中获取的 `transaction` 对象为 `uniCloud.database()` 的事务对象

## 同时发送多条数据库请求

### multiSend 合并请求

```javascript
// ❌ 不推荐: 分开请求
const banner = await db.collection('banner').get()
const notice = await db.collection('notice').get()

// ✅ 推荐: 合并请求
const bannerQuery = db.collection('banner').getTemp()
const noticeQuery = db.collection('notice').getTemp()
const res = await db.multiSend(bannerQuery, noticeQuery)
```

### 返回值结构

```javascript
// 上述请求返回以下结构
res = {
  errCode: 0,  // 请求整体执行错误码
  errMsg: "",
  dataList: [{
    errCode: 0,  // bannerQuery 对应的错误码
    errMsg: '',
    data: []     // bannerQuery 查询到的数据
  }, {
    errCode: 0,  // noticeQuery 对应的错误码
    errMsg: '',
    data: []     // noticeQuery 查询到的数据
  }]
}
```

### 在 unicloud-db 组件中使用

```html
<template>
  <view>
    <!-- 设置unicloud-db 组件为手动加载 loadtime="manual" -->
    <unicloud-db
      collection="banner"
      loadtime="manual"
      ref="udb"
      v-slot:default="{data, error}">
      <view v-if="error">{{error.message}}</view>
      <view v-else>
        <view v-for="(item,index) in data" :key="index">
          <image :src="item.url"></image>
        </view>
      </view>
    </unicloud-db>
    <button type="default" @click="test">test</button>
  </view>
</template>

<script>
export default {
  data() {
    return {
      title: 'Hello'
    }
  },
  methods: {
    test() {
      const db = uniCloud.databaseForJQL()
      const bannerQuery = this.$refs.udb.getTemp()  // 调用组件实例的getTemp方法
      const noticeQuery = db.collection('notice').getTemp()

      db.multiSend(bannerQuery, noticeQuery)
        .then(res => {
          console.log('banner', res.result.dataList[0])
          console.log('notice', res.result.dataList[1])
        })
        .catch(err => {
          console.error(err)
        })
    }
  }
}
</script>
```

### 注意事项

- HBuilderX 3.1.22及以上版本支持
- 只能批量发送查询请求
- 可以减少网络请求次数,提升性能
